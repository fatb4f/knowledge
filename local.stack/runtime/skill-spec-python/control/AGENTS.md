# Control Guide

This directory holds the human-owned policy and mapping layer for the spec-to-python workflow.

## Contents

- `mapping.cue`: normalization and naming hints between source schemas and CUE definitions
- `policy.md`: execution and review rules
- `task-contract.md`: bounded task contract and expected outputs

## Boundary Rules

- edit these files directly
- do not place generated outputs here
- keep semantic contract truth in `kernel`
